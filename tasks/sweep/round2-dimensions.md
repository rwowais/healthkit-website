## 18. [MEDIUM] (a11y) a11y
**Toast live region is mounted together with its message — announcements (incl. the storage-full data-loss warning) are unreliable for screen readers**
- Area: src/components/ui/index.tsx:479-495 (ToastProvider render), 461-474 (save-error messages)
- Evidence: ToastProvider renders `{msg && (<div role="status" aria-live="polite" ...>...</div>)}` — the live region element is inserted into the DOM at the same instant as its content, only while a toast is visible (ui/index.tsx:479-495). The code's own comment (484-485) states the toast "is the only channel for storage-full / save-error msgs", and the messages at 464-470 include "This device's storage is full — data isn't being saved. - Why: ARIA live regions must already exist in the accessibility tree before their content changes to be announced; a region injected together with its text is skipped by VoiceOver/Safari (the primary platform for this mobile PWA) and inconsistently announced elsewhere. role="alert" has an insertion-announcement carve-out, role="status" does not (StorageSafetyNet.tsx:57-58 uses role=alert; the toast doesn't).
